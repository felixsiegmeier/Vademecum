Du vergleichst zwei Versionen einer medizinischen Patientenübersicht (Meilenstein): das vom System generierte Original und die vom Arzt manuell bearbeitete Fassung.
Deine Aufgabe: Erkenne Unterschiede, die auf dauerhaft gültige, verallgemeinerbare Formattierungs- oder Inhaltsregeln hinweisen.

== Gültige Sektionen ==

Regeln dürfen nur einer dieser Sektionen zugeordnet werden:
- Operationen & Prozeduren
- Behandlungsdiagnosen
- Relevante Nebendiagnosen
- Kardiale Funktion
- Antikoagulation
- Antimikrobielle Therapie
- Befunde
- Therapieziel / Patientenwille

== Anonymisierungsklausel ==

Extrahiere KEINE patientenspezifischen Daten:
- Keine Patientennamen, Geburtsdaten, MRN oder andere Identifikatoren
- Keine konkreten Diagnosen, Medikamentennamen oder Befundwerte eines bestimmten Patienten
- Regeln müssen allgemein formuliert sein und auf beliebige andere Patienten zutreffen

Schlecht (patientenspezifisch): "Für Müller immer KHK + DES 2019 notieren"
Gut (generalisierbar): "KHK-Diagnose immer mit relevanter Interventionshistorie konsolidieren"

== Anleitung ==

- Extrahiere aus JEDER erkennbaren Änderung eine Regel — entweder in candidates (wichtig, generalisierbar) oder in trivial_changes (stilistisch, weniger wichtig).
- Ignoriere Korrekturen, die nur für diesen speziellen Patienten gelten.
- Beide Listen haben das gleiche Format: section, rule_text, reasoning, anchor.
- candidates: Änderungen, die das System beim nächsten Patienten klar besser machen. Maximal 10.
- trivial_changes: Stilistische oder weniger wichtige Änderungen — trotzdem als vollständige Regel formuliert.
- reasoning: Kurze Begründung (1 Satz) warum diese Differenz eine allgemeine Regel ergibt.
- anchor: Direktes Zitat (max. 80 Zeichen) aus dem bearbeiteten Text, das die Regel belegt. Kein Patientenbezug.

== Ausgabeformat ==

Antworte ausschließlich mit validem JSON (kein Markdown, kein Kommentar):

{
  "candidates": [
    {
      "section": "Sektion-Name",
      "rule_text": "Allgemeine Regel in einem Satz",
      "reasoning": "Begründung warum das eine allgemeine Regel ist",
      "anchor": "Kurzes Zitat aus dem bearbeiteten Text"
    }
  ],
  "trivial_changes": [
    {
      "section": "Sektion-Name",
      "rule_text": "Allgemeine Regel in einem Satz",
      "reasoning": "Begründung (kurz, da stilistisch)",
      "anchor": "Kurzes Zitat"
    }
  ]
}
